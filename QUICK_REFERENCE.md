# ⚡ Quick Reference & Troubleshooting\n\n## 📋 Quick Command Reference\n\n### First Time Setup\n```bash\n# 1. Create database in PostgreSQL\nCREATE DATABASE amazon_clone;\n\n# 2. Backend setup\ncd backend\nnpm install\nnpm run db:setup      # Creates tables and seeds data\nnpm run dev           # Start API on http://localhost:3001\n\n# 3. Frontend setup (new terminal)\ncd frontend\nnpm install\nnpm run dev           # Start UI on http://localhost:3000\n```\n\n### Daily Development\n```bash\n# Terminal 1: Backend\ncd backend && npm run dev\n\n# Terminal 2: Frontend\ncd frontend && npm run dev\n\n# Visit http://localhost:3000 in browser\n```\n\n### Database Commands\n```bash\n# Connect to database\npsql postgresql://postgres:postgres@localhost:5432/amazon_clone\n\n# View all products\nSELECT id, name, price, stock FROM products LIMIT 10;\n\n# View user's cart\nSELECT * FROM cart WHERE user_id = 1;\n\n# View all orders\nSELECT * FROM orders ORDER BY created_at DESC;\n\n# Delete all data (reset database)\nDELETE FROM cart;\nDELETE FROM orders;\nDELETE FROM order_items;\nDELETE FROM products;\nDELETE FROM users;\n\n# Reset from scratch\ndropdb amazon_clone\nCREATE DATABASE amazon_clone;  # in psql\n# Then run: npm run db:setup\n```\n\n---\n\n## 🚨 Common Problems & Solutions\n\n### ❌ \"Cannot find module 'cors'\"\n\n**Problem:** Dependencies not installed\n\n**Solution:**\n```bash\ncd backend\nnpm install\n```\n\n---\n\n### ❌ \"ECONNREFUSED\" when starting backend\n\n**Problem:** Database is not running or wrong connection string\n\n**Solution:**\n1. Make sure PostgreSQL is running:\n   - Windows: Check Services (press `Win+R`, type `services.msc`)\n   - Mac: Check System Preferences → PostgreSQL\n   - Linux: `sudo systemctl status postgresql`\n2. Check your `backend/.env` has correct DATABASE_URL\n3. Test connection:\n   ```bash\n   psql postgresql://postgres:postgres@localhost:5432/amazon_clone\n   ```\n\n---\n\n### ❌ \"Database amazon_clone does not exist\"\n\n**Problem:** Database wasn't created\n\n**Solution:**\n```bash\n# 1. Open PostgreSQL\npsql -U postgres\n\n# 2. Create database\nCREATE DATABASE amazon_clone;\n\n# 3. Exit\n\\q\n\n# 4. Setup schema and seed\ncd backend\nnpm run db:setup\n```\n\n---\n\n### ❌ API returns 503 \"Service Unavailable\"\n\n**Problem:** Backend can't reach database\n\n**Solution:**\n1. Check backend is running: `http://localhost:3001/health`\n2. Check DATABASE_URL in `backend/.env`\n3. Make sure PostgreSQL is running\n4. Check database exists: `psql -l` (lists databases)\n\n---\n\n### ❌ \"Cannot POST /cart\" or similar 404\n\n**Problem:** Endpoint doesn't exist\n\n**Solution:**\n1. Make sure backend is running (see the log message?)\n2. Call correct URL: `http://localhost:3001/cart` (not 3000)\n3. Check API is listening: `curl http://localhost:3001/health`\n\n---\n\n### ❌ Cart operations very slow\n\n**Problem:** Too many concurrent requests\n\n**Solution:**\n1. Check Network tab (F12) - see what requests are pending?\n2. Don't spam buttons - click once and wait\n3. Restart backend: `npm run dev` again\n4. Clear browser cache: `Ctrl+Shift+Delete` (or Cmd+Shift+Delete)\n\n---\n\n### ❌ \"Port 3001 already in use\"\n\n**Problem:** Something else is using that port\n\n**Solution:**\nOption A: Kill the process\n```bash\n# Windows\nnetstat -ano | findstr :3001\ntaskkill /PID <PID> /F\n\n# Mac/Linux\nlsof -i :3001\nkill -9 <PID>\n```\n\nOption B: Use different port\n```bash\n# Edit backend/.env\nPORT=3002\n\n# Edit frontend/.env.local\nNEXT_PUBLIC_API_URL=http://localhost:3002\n```\n\n---\n\n### ❌ \"Search doesn't work\" or \"Products don't load\"\n\n**Problem:** Frontend can't reach backend API\n\n**Solution:**\n1. Open browser DevTools (F12)\n2. Check Network tab - what's the error?\n3. Make sure backend is running\n4. Check `frontend/.env.local` has: `NEXT_PUBLIC_API_URL=http://localhost:3001`\n5. Restart frontend: `npm run dev` again\n\n---\n\n### ❌ \"Items disappear from cart\"\n\n**Problem:** Page refresh clears the cart (normal in demo)\n\n**Solution:**\nThis is expected! The demo doesn't persist cart to browser storage. In production, you'd use:\n- localStorage\n- sessionStorage  \n- Backend persistence (already done)\n\nThe cart IS saved on the backend (check database), but frontend shows it after page refresh.\n\n---\n\n### ❌ \"Old products showing\" after seed\n\n**Problem:** Database has old data\n\n**Solution:**\n```bash\ncd backend\nnpm run db:setup  # By design, only seeds if products table is empty\n\n# To force reseed:\npsql postgresql://postgres:postgres@localhost:5432/amazon_clone\nDELETE FROM products;\nDELETE FROM cart;\nDELETE FROM orders;\nDELETE FROM order_items;\n\\q\n\nnpm run db:setup\n```\n\n---\n\n## 🔍 Debugging Checklist\n\n### When something doesn't work:\n\n- [ ] Is backend running?
